Some browsers do not distinguish some or all of the numpad keys.

Demos:

  • Keyup or Keydown
  • Arbitrary modifiers
  • Counting combos
  • Sequential combos
  • Enforceable order

You can specify combos that include multiple non-modifer keys. This is very helpful for doing something like tile-based movement that includes diagonals with only four directional inputs.

In this demo, you can use the wasd keys to move around the tiles. You can combine two directions to move diagonally.

Sequential combos are great if you want to hide an easter egg or some kind of 'cheat code'. You can have multiple sequential combos without worrying about collisions (if one combo contained another for instance, they would both fire).

Write the word 'keypress' and the word 'JavaScript' (including proper capitalization) to see these in action.

Counting combos are a unique feature that was created for fast and predictable navigation. For this demo, the tab key is used for menu navigation. Pressing it once will always take us back to the default menu position, but we can press the spacebar while holding down the tab key to jump through the options. The same number of keypresses will always take you to the same option no matter which one is currently activated.

Why this input library?

Keypress is a very robust library with some very special features, it is easy to pick up and use, and it has a small footprint (~5kb minified). Here's some of what Keypress offers:

  • Any key can serve as a modifier show me
  • Fire on keyup and/or keydown show me
  • Special "counting" combos show me
  • Sequential combos show me

Why NOT use Keypress?

I'm still working on this, and I haven't done extensive cross-browser testing, so I don't recommend it for a production environment just yet. And I have not incorporated support for keyboard + mouse combos, which is something I will be adding in the future.
